## Developer Guidelines

1. **Dependency direction** вЂ” Domain classes must never `use` anything from Application, Port, Infrastructure, or Presentation namespaces.
2. **All classes `final`** вЂ” Every domain class must be `final readonly`. All other classes `final` unless designed for extension.
3. **No `assert()`** вЂ” Use explicit `if` + `throw` guards.
4. **No FQN in code** вЂ” Always `use` imports at the top of the file.
5. **Test doubles** вЂ” Named fakes in `tests/Support/` only. No anonymous classes. No mocks unless strictly necessary.
6. **Coverage** вЂ” Target 100%. Every defensive branch in infrastructure adapters must have a test.
7. **Commit per task** вЂ” Use task title as commit message title, task description as body.
